# 统一应用门户 - 接口设计文档

## 一、概述

### 1.1 服务划分

| 服务 | 职责 | 面向用户 | 认证方式 |
| --- | --- | --- | --- |
| Portal | 首页应用展示、应用跳转 | 所有角色 | 用户 Token |
| Console | 后台应用配置、人员管理 | 管理员 | 用户 Token |
| Server | 认证加载、开放接口 | 内部服务 / 外部业务系统 | 内部签名 / ClientId+Secret |

### 1.2 接口统计

| 服务 | 接口数 |
| --- | --- |
| Portal | 4 |
| Console | 35 |
| Server | 3 |
| **合计** | **42** |

### 1.3 通用约定

- 所有接口响应统一格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- 分页请求统一参数：`pageNum`（页码，从1开始）、`pageSize`（每页条数）
- 分页响应统一结构：`{ "total": 100, "list": [] }`
- Token 通过请求头 `Authorization: Bearer {token}` 传递
- 错误码：`401` 未认证、`403` 无权限、`200` 成功

## 二、Portal 服务

所有角色可访问，负责首页应用展示和应用跳转。

### 2.1 首页初始化

**`GET /api/portal/init`**

获取首页所需的全部数据：用户信息 + 管理员身份 + 可见应用列表。合并了原 `/api/portal/apps` 和 `/api/portal/admin-permission` 的功能，前端只需调这一个接口。

**权限：** 所有角色

**请求参数：** 无

**响应：**

```json
{
  "code": 200,
  "data": {
    "user": {
      "userId": "U001",
      "userName": "张三",
      "orgName": "南京分行",
      "deptName": "信息技术部"
    },
    "isAdmin": true,
    "isSystemAdmin": false,
    "appAdminApps": [
      { "appCode": "salary-pay", "appName": "工资代发系统" }
    ],
    "bizAdminApps": [],
    "visibleApps": [
      {
        "groupCode": "common",
        "groupName": "常用应用",
        "groupSortNo": 1,
        "apps": [
          {
            "appCode": "notice",
            "appName": "通知公告",
            "appIcon": "/icons/notice.png",
            "appDesc": "全行通知公告发布",
            "jumpUrl": "https://notice.example.com",
            "visibleType": "ALL",
            "showMenu": true,
            "showHeader": true,
            "enableWatermark": false,
            "appSortNo": 1
          }
        ]
      },
      {
        "groupCode": "biz",
        "groupName": "业务办理",
        "groupSortNo": 2,
        "apps": [
          {
            "appCode": "salary-pay",
            "appName": "工资代发系统",
            "appIcon": "/icons/salary.png",
            "appDesc": "工资代发业务办理",
            "jumpUrl": "https://salary.example.com",
            "visibleType": "ROLE",
            "showMenu": true,
            "showHeader": false,
            "enableWatermark": true,
            "appSortNo": 1
          }
        ]
      }
    ]
  }
}
```

**内部流程：**
1. 拦截器从 Redis 加载用户身份（Token → userId → identity）
2. 从 Redis 读取可见应用列表 `portal:visible:apps:{userId}`
3. 按 groupCode 聚合应用
4. 返回结果

### 2.4 应用跳转信息

**`GET /api/portal/apps/{appCode}/jump-info`**

查询应用跳转所需信息。

**权限：** 所有角色（需有该应用的访问权限）

**路径参数：**

| 参数 | 说明 |
| --- | --- |
| appCode | 应用编号 |

**响应：**

```json
{
  "code": 200,
  "data": {
    "appCode": "salary-pay",
    "appName": "工资代发系统",
    "appIcon": "/icons/salary.png",
    "appDesc": "工资代发业务办理",
    "jumpUrl": "https://salary.example.com",
    "visibleType": "ROLE",
    "showMenu": true,
    "showHeader": false,
    "enableWatermark": true
  }
}
```

> 用户在目标系统的角色由下游系统自行通过开放接口查询（见 4.2），jump-info 不返回角色信息。

**错误响应：**

- `403` — 当前用户无权访问该应用

## 三、Console 服务

后台管理，仅管理员可访问。接口按权限等级分为 A / B / C / D 四级：

| 等级 | 可访问角色 | 说明 |
| --- | --- | --- |
| A | 仅系统管理员 | 新增应用、分组管理、系统管理员管理、重置密钥 |
| B | 系统管理员 + 应用管理员（限负责应用） | 应用配置、应用管理员/业务管理员/自定义角色配置 |
| C | 系统管理员 + 业务管理员（限管理应用） | 角色人员配置 |
| D | 三种管理员（数据范围不同） | 应用列表查询 |

### 3.1 应用管理

#### 3.1.1 应用列表（D 级）

**`GET /api/admin/apps`**

**权限：** 系统管理员（全部）、应用管理员（负责应用）、业务管理员（管理应用）

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| appCode | String | 否 | 应用编号，模糊查询 |
| appName | String | 否 | 应用名称，模糊查询 |
| status | String | 否 | 状态：ENABLED / DISABLED |
| pageNum | int | 是 | 页码 |
| pageSize | int | 是 | 每页条数 |

**响应：**

```json
{
  "code": 200,
  "data": {
    "total": 15,
    "list": [
      {
        "appCode": "salary-pay",
        "appName": "工资代发系统",
        "status": "ENABLED",
        "visibleType": "ROLE",
        "createName": "管理员A",
        "createTime": "2026-04-01 10:00:00",
        "updateName": "管理员B",
        "updateTime": "2026-04-15 14:30:00"
      }
    ]
  }
}
```

**数据范围过滤：**
- 系统管理员：全部应用
- 应用管理员：`WHERE app_code IN (appAdminApps)`
- 业务管理员：`WHERE app_code IN (bizAdminApps)`

#### 3.1.3 新增/修改应用（A/B 级）

**`POST /api/admin/apps/save`**

新增或修改应用。请求体中包含 `appCode` 时为修改，不包含时为新增。

**权限：** 新增 — 仅系统管理员（A 级）；修改 — 系统管理员 + 应用管理员（限负责应用，B 级）

**请求体：**

```json
{
  "appCode": "salary-pay",
  "appName": "工资代发系统",
  "appIcon": "/icons/salary.png",
  "appDesc": "工资代发业务办理",
  "jumpUrl": "https://salary.example.com",
  "orgId": "NJ001",
  "visibleType": "ROLE",
  "showMenu": true,
  "showHeader": false,
  "enableWatermark": true,
  "enablePromotion": false,
  "sortNo": 1
}
```

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| appCode | 新增必填 | 有值为修改，无值为新增。修改时不建议变更 |
| appName | 是 | 应用名称 |
| jumpUrl | 是 | 跳转地址 |
| visibleType | 是 | ALL / ROLE |
| 其他字段 | 否 | 见应用详情字段 |

**响应：** 标准成功/失败

**缓存失效：** 修改后失效 `portal:visible:apps:*`（可见范围可能改变）

## 四、Server 服务

负责认证初始化和给外部业务系统提供开放接口。

### 4.1 认证初始化（内部接口）

**`POST /internal/auth/init`**

Portal / Console 首次遇到未缓存的 Token 时调用此接口。Server 验证 Token、加载用户全部权限、写入 Redis。

**认证方式：** 内部签名（`X-Internal-App` + `X-Internal-Timestamp` + `X-Internal-Sign`）

**请求头：**

| Header | 说明 |
| --- | --- |
| X-Internal-Token | 用户 Token 值 |

**响应：**

```json
{
  "code": 200,
  "data": {
    "userId": "U001",
    "userName": "张三",
    "orgId": "NJ001",
    "orgName": "南京分行",
    "deptId": "D001",
    "deptName": "信息技术部",
    "isAdmin": true,
    "isSystemAdmin": true,
    "appAdminApps": [
      { "appCode": "salary-pay", "appName": "工资代发系统" }
    ],
    "bizAdminApps": [],
    "visibleApps": [
      {
        "appCode": "salary-pay",
        "appName": "工资代发系统",
        "appIcon": "/icons/salary.png",
        "jumpUrl": "https://salary.example.com",
        "visibleType": "ROLE",
        "showMenu": true,
        "showHeader": false,
        "enableWatermark": true,
        "groupCode": "biz",
        "groupName": "业务办理",
        "groupSortNo": 2,
        "appSortNo": 1
      }
    ]
  }
}
```

**内部流程：**
1. 调外部身份平台验证 Token → 获取 userId、userName、orgId、deptId
2. 查 system_admin 表 → isSystemAdmin
3. 查 app_admin 表 → appAdminApps
4. 查 biz_admin 表 → bizAdminApps
5. 执行可见应用 SQL → visibleApps
6. 计算 isAdmin = isSystemAdmin || appAdminApps非空 || bizAdminApps非空
7. 写入 Redis 3 个 Key（TTL 30 分钟）
8. 返回完整数据

---

### 4.2 查询用户角色（开放接口）

**`GET /api/open/apps/{appCode}/users/{userId}/roles`**

给目标业务系统调用，查询指定用户在某个应用下的自定义角色。

**认证方式：** ClientId + ClientSecret

**请求头：**

| Header | 说明 |
| --- | --- |
| X-Client-Id | 应用 ClientId |
| X-Client-Secret | 应用 ClientSecret |

**路径参数：**

| 参数 | 说明 |
| --- | --- |
| appCode | 应用编号（必须等于 Client-Id） |
| userId | 用户ID |

**响应：**

```json
{
  "code": 200,
  "data": {
    "appCode": "salary-pay",
    "userId": "U001",
    "roles": [
      { "roleCode": "salary-pay_operator", "roleName": "经办岗" },
      { "roleCode": "salary-pay_checker", "roleName": "审核岗" }
    ]
  }
}
```
