# 文档入口

当前维护的核心文档如下：

| 文档 | 面向对象 | 用途 |
| --- | --- | --- |
| `需求文档.md` | 产品、研发、测试 | 业务范围、角色、功能需求 |
| `技术设计文档.md` | 后端、架构 | 系统架构、模块职责、核心设计 |
| `接口文档.md` | 前后端、测试 | API 契约、请求响应、状态码 |
| `前端对接文档.md` | 前端 | 页面、接口调用、认证跳转、权限展示 |
| `认证设计.md` | 前端、后端、测试 | 上游登录、本系统内部认证、续期、下游规划 |
| `sql.txt` | 后端、DBA | 数据库建表脚本 |

归档文档位于 `archive/`，用于保留历史分析和阶段性沟通材料，默认不再维护：

| 归档文档 | 说明 |
| --- | --- |
| `archive/上游系统认证对接流程.md` | 已合并进 `认证设计.md` |
| `archive/下游系统认证对接设计.md` | 已合并核心规划进 `认证设计.md` |
| `archive/认证流程全链路分析.md` | 已合并当前链路进 `认证设计.md` |
| `archive/request-call-chain.md` | 已合并关键调用链进 `技术设计文档.md` 和 `认证设计.md` |
| `archive/上游认证前后端分工.html` | 阶段性沟通材料，当前以 `认证设计.md` 和 `前端对接文档.md` 为准 |
| `archive/项目框架说明.md` | 框架说明已并入 `技术设计文档.md` |

## 推荐阅读顺序

```text
产品/测试：需求文档.md -> 接口文档.md -> 认证设计.md
前端：前端对接文档.md -> 接口文档.md -> 认证设计.md
后端：技术设计文档.md -> 认证设计.md -> 接口文档.md
```
