上游系统认证对接前后端分工

适用场景：本系统作为“南京分行应用管理系统”接入上游统一认证平台，采用授权码模式完成单点登录。

一、结论先行

上游认证对接不是单纯的前端跳转，也不是单纯的后端接口调用，而是前后端配合完成的一条登录闭环。

核心分工： 前端负责识别未登录状态、发起登录入口、处理页面跳转体验；后端负责拼接上游认证地址、接收授权码、换取 token、解析用户、建立本系统登录态和权限上下文。
角色 主要职责 不建议承担的职责
前端前端页面 请求初始化接口、发现 401 后跳登录、展示登录中/失败提示、登录成功后回到原页面。 不保存上游 accessToken / refreshToken，不拼 client_secret，不直接调用上游 token 接口。
后端本系统后端 302 跳上游、生成 state、callback 换 token、保存 session、刷新 token、恢复 UserContext。 不校验用户密码，不直接判断上游是否已登录，这由上游认证服务负责。
共同接口约定 统一 401 处理、回跳地址、登录成功落点、登出行为、异常提示。 不要让前后端各自维护一套登录状态判断规则。

二、整体登录流程

默认入口约定： 上游点击“南京分行应用管理系统”后默认进入门户首页 /，前端默认调用 GET /api/portal/init。 如果未登录，前端跳转 /api/auth/login?redirect=/。 上游注册的 redirect_uri 仍是 /api/auth/callback，不要和登录成功后的前端页面混淆。
1. 访问系统 用户打开本系统首页或刷新页面。
2. 初始化 前端调用本系统初始化接口。
3. 未登录 后端返回 401，或页面访问被重定向。
4. 跳上游 浏览器访问本系统登录入口，由后端 302 到上游。
5. 上游认证 上游判断 Cookie/Session，必要时展示登录页。
6. 回调换票 上游带 code 回跳，本系统后端换 token。
7. 进入系统 后端建立本系统 session，前端重新初始化。

推荐交互链路

用户访问前端页面（默认 /）
  -> 前端调用 GET /api/portal/init
  -> 如果返回 200，正常进入系统
  -> 如果返回 401，跳转 GET /api/auth/login?redirect=/（默认首页）
  -> 后端 302 到上游认证地址
  -> 上游登录成功后回跳 GET /api/auth/callback?code=xxx&state=xxx
  -> 后端 code 换 token，建立本系统 Cookie/Session
  -> 后端 302 回 redirect 指定页面（默认 /）
  -> 前端重新调用 GET /api/portal/init

四、前端要做哪些

前端1. 页面初始化

  • 页面加载时调用 GET /api/portal/init 获取当前用户、权限和可见应用。
  • 如果接口成功，按返回数据渲染页面。
  • 如果接口返回 401，说明本系统未登录或登录态失效，需要跳转登录入口。

前端2. 统一处理 401

  • 在请求封装层统一拦截 401。
  • 记录当前页面路径，作为登录成功后的回跳目标。
  • 跳转到 /api/auth/login?redirect=当前页面。
  • 避免多个接口同时 401 时重复跳转。

前端3. 登录成功后的页面恢复

  • callback 完成后，后端会 302 回前端页面。
  • 前端重新调用初始化接口，刷新用户信息和权限。
  • 如果原页面需要权限，按最新权限判断是否展示。

前端4. 登出入口

  • 提供退出按钮，调用 POST /api/auth/logout。
  • 退出成功后清理前端内存状态。
  • 跳转登录页或系统未登录页。

前端5. 异常提示

  • 登录超时：提示“登录已超时，请重新登录”。
  • 无权限：展示无权限页面，不要反复跳登录。
  • 认证服务异常：展示系统繁忙或认证失败提示。

禁止前端不要做的事

  • 不要保存上游 accessToken 或 refreshToken 到 localStorage。
  • 不要持有 client_secret。
  • 不要直接调用上游 token 接口。
  • 不要自己解析上游 token 作为登录依据。

五、后端要做哪些

后端1. 登录入口

新增接口：

GET /api/auth/login?redirect=/console/apps
  • 生成随机 state。
  • 把 state 和 redirect 写入 Redis，TTL 建议 5 到 10 分钟。
  • 拼接上游认证地址。
  • 302 重定向到上游认证服务。

后端2. callback 回调

新增接口：

GET /api/auth/callback?code=xxx&state=xxx
  • 校验 state 是否存在、是否一致、是否过期。
  • 使用 code 调上游 token 接口。
  • 拿到 accessToken、refreshToken、expiresIn。
  • 解析 token 或调用用户信息接口获取用户基础信息。
  • 加载本系统权限和可见应用。
  • 建立本系统 Cookie/Session。
  • 302 回前端原访问页面。

后端3. 本系统登录态

  • 生成随机 PORTAL_SESSION。
  • Cookie 使用 HttpOnly、Secure、SameSite=Lax。
  • Redis 保存 session、用户信息、token 和过期时间。
  • TTL 不超过上游 accessToken 剩余有效期。

后端4. 认证拦截器

  • 从 Cookie 中读取 PORTAL_SESSION。
  • 从 Redis 恢复用户身份和权限。
  • 写入 UserContext。
  • 请求结束后清理 UserContext。
  • 未登录返回 401，接口请求不建议直接返回 HTML 登录页。

后端5. token 续期

  • 每次请求检查 accessToken 是否即将过期。
  • 剩余有效期小于 10 分钟时，用 refreshToken 续期。
  • 续期成功后更新 Redis 和 Cookie Max-Age。
  • 续期失败时清理本系统登录态，返回 401 或重走登录。

后端6. 登出

新增接口：

POST /api/auth/logout
  • 删除 Redis session。
  • 清除浏览器 Cookie。
  • 如上游提供统一登出接口，再按需求接入。

六、接口约定建议

接口 调用方 职责 返回/行为
GET /api/portal/init 前端 获取当前用户、管理员标识、可见应用。 已登录返回 200；未登录返回 401。
GET /api/auth/login?redirect=xxx 前端浏览器跳转 后端生成 state，并跳转上游认证服务。 302 到上游认证地址。
GET /api/auth/callback?code=xxx&state=xxx 上游认证服务回跳 后端校验 state、code 换 token、建立本系统 session。 成功后 302 回前端页面；失败时返回错误页或重新登录。
POST /api/auth/logout 前端 退出本系统登录。 清理 Cookie/Redis，返回 200。
后端自动续期 后端过滤器内部处理 使用 refreshToken 刷新 accessToken。 当前不暴露 POST /api/auth/refresh 给前端。

七、状态码和前端处理

状态 含义 前端处理 后端处理
200 已登录且有权限。 正常渲染。 返回业务数据。
401 本系统未登录或登录态失效。 跳转 /api/auth/login。 不要返回业务数据。
403 已登录但没有本系统权限。 展示无权限页面。 返回明确错误信息。
500 认证服务或系统异常。 展示失败提示，允许重试。 记录日志，避免泄露敏感信息。
重要约定： 401 表示“需要重新登录”，403 表示“登录了但没有权限”。前端不要把 403 当成 401 反复跳登录。

八、前后端联调清单

前端准备
统一请求拦截器、401 跳转逻辑、退出按钮、无权限页面、登录失败提示页面。
后端准备
登录入口、callback、上游 token client、session 存储、认证拦截器、refreshToken 续期、登出接口。
共同确认
登录成功后跳转路径、401/403 规则、callback 失败展示方式、UAT 和生产 redirect_uri。
风险关注
不要暴露 client_secret，不要把上游 token 存 localStorage，state 要短 TTL 且一次性使用。

联调用例

  1. 首次访问本系统，上游无登录态，应展示上游登录页。
  2. 首次访问本系统，上游已有登录态，应直接回跳并进入系统。
  3. callback 缺少 code，应失败并提示。
  4. callback state 过期，应提示登录超时或重新登录。
  5. 本系统 session 过期、上游仍有效，应静默重新建本系统登录态。
  6. accessToken 即将过期，应自动 refresh 成功。
  7. refreshToken 过期，应清理本系统登录态并重新跳上游。
  8. 用户已登录但无权限，应返回 403 并展示无权限页。
  9. 退出登录后再次访问，应重新走登录流程。
