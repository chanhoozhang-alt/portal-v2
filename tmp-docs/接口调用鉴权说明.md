# 接口调用鉴权说明

## 介绍

面向对象：接口调用方

如果需要通过一事通鉴权体系调用第三方应用接口，为了方便应用方进行接口调用，提供了如下工具包版本和非工具包版本：

| 方式 | 适用场景 |
| :--- | :--- |
| 工具包版本（ZA21 项目） | 建议对接应用优先使用 ZA21 版本的工具包 |
| 工具包版本（非 ZA21 项目） | 适用于行内 Java 类应用，只需配置参数，具体实现由工具包集成 |
| 非工具包版本 | 适用于行外、非 Java 类应用（无法引用行内 maven 仓库），需自行实现整套鉴权方法 |

---

## 使用 ZA21 工具包调用接口说明

### 功能说明

使用工具包帮助进行接口调用，直接引入工具包即可。

### 第一步：Java 项目引用工具包

SpringBoot 项目添加以下依赖：

```xml
<parent>
    <groupId>ZA21</groupId>
    <artifactId>bee-starter-parent</artifactId>
    <version>最新版本</version>
</parent>

<dependencies>
    <dependency>
        <groupId>ZA21</groupId>
        <artifactId>bee-starter-auth</artifactId>
    </dependency>
</dependencies>
```

> 最低版本要求：**55.5.0-RELEASE**
>
> 建议版本：5.10.21

### 第二步：添加 yml 配置

#### 工具包版本 < 5.9.0

```yaml
bee:
  auth:
    toolkit:
      clientId: # 注册应用的 clientId
      privateKey: # 使用默认国密算法提供的应用私钥（应用注册提交的公钥对应的私钥）
      centerPublicKey: # 使用默认国密算法提供的认证中心公钥，用于验证 jwt（固定值见《token获取》文档）
      # rsaPrivateKey: # 使用 RS256 算法提供的应用私钥，和 privateKey 二选一
      # centerRsa2048PublicKey: # 使用 RS256 算法提供的认证中心公钥，用于验证 jwt
      centerUri: http://one-account-gateway.paasuat.cmbchina.cn/auth-server # 测试环境地址
      redirectUri: # 对接登录才需要配置（和注册应用的回调地址保持一致）
    slow-request-mills: # SignClient 慢请求阈值（毫秒），超过打印 Warn 日志，默认 5000
    time-log: # 慢请求日志打印开关，默认 false
    connection-request-timeout: # 从连接池获取连接的超时时间（毫秒），默认 10000
    connection-timeout: # 与服务器建立连接的超时时间（毫秒），默认 10000
    socket-timeout: # 等待数据返回的超时时间（毫秒），默认 60000
    max-conn-total: # 连接池最大连接数，默认 20
    max-per-route: # 每个 Route 的最大连接数，默认 20
```

#### 工具包版本 >= 5.9.0

> 若不需要工具包自动登录拦截功能，需配置 `bee.auth.login.filterPatterns` 为系统中不存在的 url（如 `/disable-login-filter`）。

```yaml
bee:
  auth:
    # toolkit 层级在新版本中建议去掉，如保留需将 toolkit 之后的所有层级往后移动一层
    clientId: # 注册应用的 clientId
    privateKey: # 国密算法应用私钥（注册提交的公钥对应的私钥）
    publicKey: # 国密算法应用公钥（注册提交的公钥）
    centerPublicKey: # 国密算法认证中心公钥，用于验证 jwt（固定值见《token获取》文档）
    # rsaPrivateKey: # RS256 算法应用私钥，和 privateKey 二选一
    # rsaPublicKey: # RS256 算法应用公钥，和 publicKey 二选一
    # centerRsaPublicKey: # RS256（512 位）认证中心公钥
    # centerRsa2048PublicKey: # RS256（2048 位）认证中心公钥
    centerUri: http://one-account-gateway.paasuat.cmbchina.cn/auth-server # 生产环境请使用 http
    slowProcessLog: # 是否打印潜在耗时操作的时间
    slowProcessMills: # 慢处理阈值（毫秒），超过打印 warn 日志
    login:
      authorizeUri: # 认证中心授权地址
      redirectUri: # 对接登录才需要配置（和注册应用的回调地址保持一致）
      logoutRedirectUri: # 登出成功后的回跳地址
      filterPatterns: # loginFilter 拦截路径，默认 "/**"（若不需要自动对接认证服务，请配置为不存在的路径）
      filterSkipPatterns: # loginFilter 需要排除的拦截路径
      filterOrder: # loginFilter 加载顺序，默认 -1
      tokenExpireminutes: # token 过期时间（分钟），默认 36
    api:
      filterPatterns: # api 拦截路径（仅服务端应用需要配置）
      filterOrder: # api 拦截器顺序（仅服务端应用需要配置）
    httpclient:
      slow-request-mills: # 慢请求阈值（毫秒），默认 5000
      connection-request-timeout: # 从连接池获取连接的超时时间（毫秒），默认 10000
      connection-timeout: # 与服务器建立连接的超时时间（毫秒），默认 10000
      socket-timeout: # 等待数据返回的超时时间（毫秒），默认 60000
      max-conn-total: # 连接池最大连接数，默认 20
      max-per-route: # 每个 Route 最大连接数，默认 20
```

> **注意**：`connection-request-timeout`、`connection-timeout`、`socket-timeout`、`max-conn-total`、`max-per-route` 参数请根据业务情况合理设置。

### 第三步：在启动类上添加注解

添加 `@EnableAuthClient` 注解自动注入相关 Bean，主要提供两个 Bean：

- **TokenHelper**：用于获取 accessToken
- **SignClient**：用于发送 http 请求并自动加签

```java
package com.demo;

import com.cmb.bee.auth.client.config.EnableAuthClient;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
@EnableAuthClient
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

### 第四步：对接接口加签

目前工具包支持两种方式：

1. **FeignInterceptor**：对 Feign 调用加签
2. **SignClient**：对 Apache HttpClient 的封装，自动对请求加签

#### a. 获取 Token

##### 授权码模式（对接登录时使用）

```java
public class TokenTest {
    @Autowired
    private TokenHelper tokenHelper;

    @Test
    public void getToken() {
        OAuthToken oAuthToken = tokenHelper.getTokenByAuthorizationCode("认证通过返回的授权码字符串");
        String idToken = oAuthToken.getIdToken(); // idToken 中为用户信息
    }
}
```

##### 客户端模式

```java
public class TokenTest {
    @Autowired
    private TokenHelper tokenHelper;

    @Test
    public void getToken() {
        OAuthToken oAuthToken = tokenHelper.getClientToken();
        String accessToken = oAuthToken.getAccessToken(); // 用于后续调用其他接口的凭据
    }
}
```

#### b. 调用 OpenAPI（两种方式任选其一）

##### 方式一：@Feign

工具包提供了两个 FeignInterceptor：

- **SignInterceptor**：对请求加签
- **TokenInterceptor**：继承 SignInterceptor，在加签的基础上同时获取 token

**工具包版本 < 5.9.0**

```java
// AuthProperties 已被自动注入为 Bean，可直接使用
@Bean
public DemoClient demoClient(AuthProperties authProperties) {
    return Feign.builder()
            .decoder(new JacksonDecoder())
            .encoder(new JacksonEncoder())
            .requestInterceptor(new TokenInterceptor(authProperties))
            .target(DemoClient.class, baseUri); // baseUri 为调用目标接口的根地址
}
```

**工具包版本 >= 5.9.0**

```java
// AuthProperties、SignClient 已被自动注入为 Bean，可直接使用
@Bean
public DemoClient demoClient(AuthProperties authProperties, SignClient signClient) {
    return Feign.builder()
            .decoder(new JacksonDecoder())
            .encoder(new JacksonEncoder())
            .requestInterceptor(new TokenInterceptor(authProperties, signClient))
            .target(DemoClient.class, baseUri); // baseUri 为调用目标接口的根地址
}
```

##### 方式二：@ApiHelper

`ApiHelper` 是对 `SignClient` 和 `TokenHelper` 的封装。调用平台开放 API 除了需要加签外，还需要将 token 附加在请求头中（由 TokenHelper 帮助获取）。

```java
package com.oauth.client.service;

import com.cmb.oa.one.account.oauth.client.http.ApiHelper;
import com.cmb.oa.one.account.oauth.client.model.param.QueryParams;
import com.cmb.oa.one.account.oauth.client.sign.constant.HttpMethod;
import lombok.extern.slf4j.Slf4j;
import org.apache.http.entity.ContentType;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;

@Service
public class OpenApiService {

    @Value("${api.server.uri}")
    private String apiBaseUri; // OpenAPI 根地址

    private final ApiHelper apiHelper; // 启动类加 @EnableAuthClient 后可直接注入

    public OpenApiService(ApiHelper apiHelper) {
        this.apiHelper = apiHelper;
    }

    public String getUsers(String viewId) {
        String apiUri = String.format("%s/v2/users?viewId=%s", apiBaseUri, viewId);
        QueryParams queryParams = new QueryParams().pageIndex(0);
        queryParams.where("openId", "3C05E97CE725671C89F7A594A60700A6");
        // 带请求 body 的 POST 请求
        return apiHelper.execute(HttpMethod.POST, apiUri, queryParams, ContentType.APPLICATION_JSON);
        // 不带 body 的 GET 请求：
        // return apiHelper.execute(HttpMethod.GET, apiUri);
    }
}
```
